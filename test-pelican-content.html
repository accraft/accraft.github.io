This is my first attempt at making some content for my github.io page. Nice that all content is in markdown, which I have some limited experience. I make most of my content in Python, developing stuff in Notepad++, and R/R Studio, so I hope to integrate some of that stuff within Pelican. Especially the dynamic charts created using plotly look pretty slick.

Right now, I have two separate projects that create this content. 1. The actual github.io page which only contains content, readme, and gitignore. 2. A project where I have Pelican installed which actually creates content. Luckily keeping the two pages in sync seems fairly easy, once I run the pelican content command, I run rsync using wsl, wsl rsync -a ./pelican_gitio/output/ ./accraft.github.io.
